Glossary

The compressed essence. Every term is tagged with its bucket from Lesson 01's map and pinned to a real spot in your FHA project. Print it; keep it beside you.

Browser ③ what the browser runs Author ① what you write Toolchain ② the build tools Concept an idea, not a thing

HTML Browser ③

HyperText Markup Language — the structure of a page (headings, paragraphs, buttons, images). The nouns.

In FHA index.html

Element Browser ③

The one repeating shape of HTML: content wrapped in a matching pair of tags (e.g. <p>…</p>). Elements nest inside elements, making the page a tree.

In FHA <div id="root"> in index.html

Tag Browser ③

The marker that opens or closes an element: <p> opens, </p> closes. The name in the tag decides what kind of element it is.

In FHA <title>, <body>, <script>

Attribute Browser ③

A name="value" setting inside an opening tag that configures the element — like a keyword argument to a function call.

In FHA id="root", src="/src/main.tsx"

head vs body Browser ③

<head> = machine-facing info about the page (title, settings, links to CSS) — invisible. <body> = everything the visitor actually sees.

In FHA the two halves of index.html

CSS Browser ③

Cascading Style Sheets — the look: colour, spacing, fonts, layout. Styles the HTML nouns.

In FHA styles/global.css, Button.css

JavaScript (JS) Browser ③

The only programming language a browser runs. Provides behaviour — responding to clicks, changing the page over time.

In FHA the compiled output of every .tsx file

DOM Concept

Document Object Model — the live, in-memory tree of the page that JavaScript reads and changes. "Changing the page" means changing the DOM.

In FHA what React updates for you under the hood

React Author ①

A library for building UIs out of reusable components. You describe what the screen should look like; React keeps the DOM in sync.

In FHA every .tsx file; react in package.json

Component Concept

A reusable, self-contained piece of UI — a function that returns a chunk of page. Defined once, used many times.

In FHA src/components/Button/Button.tsx

Props Concept

The inputs you pass into a component — written like HTML attributes, received like function arguments. children is the special prop holding the content between the tags.

In FHA <Button variant="accent">Get involved</Button>

JSX / TSX Author ①

HTML-looking syntax written inside JavaScript/TypeScript. The build step compiles it to plain JS. .tsx = TypeScript + JSX.

In FHA the <Button>…</Button>-style markup in any .tsx

TypeScript (TS) Author ①

JavaScript plus a type checker that flags mismatches before the code runs. A "superset" — all JS is valid TS. Compiled down to JS.

In FHA .ts/.tsx files; tsconfig.json

Library vs Framework Concept

A library is code you call when you want (three.js, Lenis). A framework is code that calls you — it sets the structure and you fill in the blanks. React sits in between; people use both words for it.

In FHA three & lenis are libraries

npm Toolchain ②

Node Package Manager — downloads and version-tracks third-party libraries. Your pip / install.packages() for the web.

In FHA npm install, npm run dev

package.json Toolchain ②

The project's recipe: its name, its scripts (dev, build), and exactly which libraries + versions it depends on.

In FHA lists react, three, lenis, vite…

node_modules/ Toolchain ②

The folder where npm dumps every downloaded library. Huge, generated, never edited by hand, never committed to git. Your virtual-env library folder.

In FHA created by npm install

Build tool / Bundler Toolchain ②

Translates author-layer code (React/TS/JSX) into the plain HTML/CSS/JS the browser runs, and bundles many files into few. The compiler of the web.

In FHA Vite

Vite Toolchain ②

The specific build tool here. npm run dev = live server that recompiles on save; npm run build = optimised output for release.

In FHA vite.config.ts

Dev server / Hot reload Concept

A local server that serves your site while you work and refreshes the instant you save — like re-running a notebook cell automatically.

In FHA npm run dev (Vite, opens the site)

dist/ (build output) Concept

The folder of final, compiled, browser-ready files produced by npm run build. This is what actually gets deployed.

In FHA dist/ after a build

CSS custom property / Design token Browser ③

A reusable named value in CSS (--name: value) — a variable for colours, spacing, fonts. "Design tokens" = the branded set of them.

In FHA the --fha-* values in styles/global.css

Front end vs Back end Concept

Front end = everything running in the visitor's browser (this whole map). Back end = a server + database storing/serving data. FHA is front-end only.

In FHA no back end — it's a marketing site
