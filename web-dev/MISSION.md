# Mission: Web Development Literacy

## Why
Amir vibe-codes web projects (like the FHA website) but can't see what the AI is
actually building — which leaves him unable to judge, steer, or fix the output.
The goal is to become literate enough in modern web development to **read and
make small, safe edits to the code**, and to know **when to use what and why** —
not to write production code from scratch.

## Success looks like
- Open any file in a web project and correctly say what layer it belongs to and what it does.
- Read an AI's plan for a web task and spot when it's reaching for the wrong tool.
- Make a small, confident edit (change text, a colour, a layout value) and predict the result.
- Explain, in plain terms, what HTML, CSS, JavaScript, React, TypeScript, npm and a build tool each do.

## Constraints
- Comes from MATLAB / R / Python (strong) — thinks like a data scientist / ML engineer, not a web dev.
- Time-boxed: short lessons, completable quickly. No syntax memorisation drills.
- Learns best **diagrams & maps first**, detail second.
- Examples anchored in his real **FHA website** project, with occasional clean outside examples.

## Out of scope (for now)
- Backend / servers / databases (the FHA site is frontend-only — revisit later if needed).
- Writing apps from scratch; framework deep-dives beyond React basics.
- DevOps, deployment pipelines, SEO, testing frameworks (name them on the map, don't drill in yet).
