# Web-Dev Literacy — my learning workspace

This is a self-contained **learning workspace**, not tied to any code project.
Everything I'm learning about web development lives here as files — so it survives
no matter what happens to any chat or any project folder.

## How to resume a lesson
1. Open this folder (`learning\web-dev\`) in Claude Code (or VS Code with Claude).
2. Run `/teach`. Claude reads `MISSION.md` + `learning-records/` and picks up where we left off.
3. Or just say: *"continue my web dev teaching."*

## What's here
| Folder / file | What it is |
|---|---|
| `MISSION.md` | Why I'm learning this + what success looks like. The compass. |
| `lessons/` | The lessons themselves — open the `.html` files in a browser. Start at `0001`. |
| `reference/glossary.html` | Authoritative glossary. Print it; keep it handy. |
| `RESOURCES.md` | The trusted external resources behind the lessons. |
| `learning-records/` | What I've actually learned (drives what comes next). |
| `assets/` | Shared styles + widgets the lessons use. |
| `NOTES.md` | Teaching preferences + working notes. |

## Start here
→ Open **`lessons/0001-the-map-of-web-development.html`** in your browser.

> Note: this folder lives in **Downloads**, which isn't backed up and can get
> auto-cleaned. To keep it safe long-term, move the whole `learning\` folder into
> OneDrive (or push it to a private GitHub repo). It's just files — moving is safe.
