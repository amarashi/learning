# Established: strong general programming, zero web-platform model

Amir is fluent in MATLAB / R / Python (data analysis + ML, professionally). He
reasons well about code, packages, environments, and compile/interpret pipelines —
analogies to those land hard and should be used. He has shipped several
AI-"vibe-coded" web projects (incl. the FHA site) but had **no mental model of the
web stack** going in: couldn't say what HTML/CSS/JS, React, TypeScript, npm, or a
build tool each do, or how they relate.

**Why it matters for future sessions:** Teach the *why/when/relationship*, not
syntax. Don't re-explain general programming concepts (variables, functions, loops,
packages) — he has those. Do explain anything web-platform-specific (the DOM, the
browser-runs-only-3-languages model, the build step, JSX). Start every concept from
the [[MISSION.md]] map of three buckets (browser ③ / author ① / toolchain ②).

**Mission set this session** (see [[MISSION.md]]): web-dev *literacy* — read & tweak
code, know when to use what and why; not write from scratch. Anchor in the real FHA
project; diagrams/maps first. Backend is out of scope until a project needs it.

Lesson 01 (the map) was *delivered*, not yet *demonstrated* — no evidence of
retention yet. Next session: confirm the three-bucket model stuck (have him sort a
few unfamiliar terms), then drill into one bucket — likely Layer ③ HTML via
`index.html`, since "read & tweak" starts with reading structure.
