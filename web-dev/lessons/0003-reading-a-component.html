Lesson 03 · Layer ① — React & JSX

Reading a Component

Last lesson ended on a cliffhanger: your index.html is nearly empty because the page lives in .tsx files. This is where we go find it.

A React app is a tree of components. Learn to read one component — a function that returns HTML-shaped markup — and the whole src/ folder stops being a mystery.

The line that fills the empty box

Remember the <div id="root"> and the <script src="/src/main.tsx">? Here is that main.tsx, in full — the bridge from HTML into React:

import { createRoot } from 'react-dom/client'
import { LandingPage } from './LandingPage'

const container = document.getElementById('root')   find the empty box

createRoot(container).render(
  <LandingPage />                                draw this into it
)

your file · src/main.tsx (lightly trimmed)

That's the entire handoff: grab the root div, render <LandingPage /> inside it. Notice <LandingPage /> is written like an HTML tag — but there is no such HTML tag. It's your own component, used like a tag. That's the central trick of React.

What a "tree of components" means

LandingPage doesn't draw everything itself — it's made of smaller components, which are made of smaller ones still, until at the very bottom you finally hit real HTML tags. This is the actual shape of your site:

  • <LandingPage> your whole page
    • <Header>
    • <FutureHealthHero> the 3-D hero
    • <Section>
      • <Heading>
      • <Stat> a metric
      • <Button>
        • <button> ← finally, a real HTML tag
your component (Capitalised) — Author ① built-in HTML tag (lowercase) — Browser ③
The single most useful reading rule Capitalised tag = a component you (or a library) wrote — go find its .tsx file to see inside. lowercase tag = a built-in HTML element — it bottoms out in the browser. <Button> is yours; <button> is the browser's.

Inside one component

A component is just a function that returns markup. Here's the heart of your real Button, with the TypeScript noise stripped away:

// a function, named with a Capital letter
function Button({ variant, children }) {
  return (
    <button className={`fha-button fha-button--${variant}`}>
      {children}
    </button>
  )
}

your file · src/components/Button/Button.tsx (simplified)

Three things to see, and only three:

  • It returns JSX — that HTML-shaped markup inside the return. JSX is HTML living inside JavaScript. Two tiny spelling differences from real HTML: it's className (not class), and every tag must close.
  • The curly braces { } are an escape hatch back into JavaScript. Outside them you're writing markup; inside them you're writing code — a variable, a calculation, anything. {children} means "drop whatever was passed in, right here."
  • { variant, children } are its props — the inputs to the function (see below).

Props: a component's inputs

Props are to a component what attributes are to an HTML tag — and what arguments are to a function. You pass them the same way you'd write attributes:

<Button variant="accent">Get involved</Button>

Here variant="accent" is a prop (make it berry-pink), and Get involved — the content between the tags — arrives inside the component as children. Change "accent" to "primary" and the same button turns navy. That is a small, safe, predictable tweak — exactly your mission.

Maps to what you already know A component is a function that returns a figure; props are its arguments. <Button variant="accent">Get involved</Button> is just Button(variant="accent", children="Get involved") in disguise. And the JSX { } is string interpolation — R's glue() or a Python f-string — splicing a live value into otherwise-static markup.
What we're deliberately ignoring (for now) Real Button.tsx also has interface ButtonProps, default values like variant = 'primary', {...rest}, and a cx(…) helper. That's TypeScript typing and plumbing — important later, but pure noise for reading today. Skip past it to the return and you'll understand what the component renders.
Your win this lesson You can open any .tsx file, scroll to its return, and read the JSX: tell your components (Capitalised) from HTML tags (lowercase), spot the props being passed, and know that { } means "real JavaScript here." You can now trace your page from root down to the markup.

Quick check — recall only, no clues in the wording:

1. You see <Section> in a .tsx file. What does the capital S tell you?

2. In <Button variant="accent">, what is variant="accent"?

3. Inside JSX you see {`fha-button--${variant}`}. What do the curly braces mean?
