# Resources

High-trust resources backing the lessons. Verified live as of 2026-06-25.

## Primary references (the canon)

- **MDN — Learn Web Development** — https://developer.mozilla.org/en-US/docs/Learn_web_development
  The gold-standard, free, Mozilla-maintained curriculum (updated Aug 2025). Covers
  how the web works, HTML, CSS, JavaScript, accessibility, React fundamentals.
  *Trust: highest.* Our default citation for HTML/CSS/JS facts.

- **MDN Web Docs (reference)** — https://developer.mozilla.org/
  The dictionary for every HTML element, CSS property, and JS function. Reference, not tutorial.

- **react.dev — Learn React (Quick Start)** — https://react.dev/learn
  Official React docs. "80% of the React concepts you use daily." Our default for React.

- **Vite — Why Vite / Guide** — https://vite.dev/guide/why.html
  Official docs for the build tool used in the FHA project. Explains the dev-server +
  build-step model.

- **TypeScript Handbook** — https://www.typescriptlang.org/docs/handbook/intro.html
  Official docs for TypeScript (the `.ts`/`.tsx` files). For when we reach types.

## Orientation / maps

- **roadmap.sh — Frontend** — https://roadmap.sh/frontend
  The famous (overwhelming) "everything a frontend dev could learn" map. Useful as a
  checklist of what exists — NOT a study order. Top-level groups: HTML/CSS/JS,
  TypeScript, version control (Git), responsive design & UX, accessibility, APIs,
  testing & debugging, performance, frameworks (React/Vue), SEO.
  *Likely the "list" Amir found.*

- **web.dev (Google)** — https://web.dev/learn
  Best-practice guides on modern web (performance, CSS, accessibility). High trust.

## Communities (for wisdom / asking real people)
- **r/webdev** — https://www.reddit.com/r/webdev/ — broad, beginner-friendly.
- **r/learnprogramming** — https://www.reddit.com/r/learnprogramming/ — gentle, good for "why" questions.
- **MDN / Mozilla Discord & community channels** — linked from the Learn pages.
