# Graphic Design — training the eye

A self-contained **learning workspace** (sibling of `../web-dev/`), not tied to any
project. Everything I'm learning about graphic design lives here as files — so it
survives no matter what happens to any chat or folder.

## How to resume a lesson
1. Open this folder (`learning\graphic-design\`) in Claude Code (or VS Code + Claude).
2. Run `/teach`. Claude reads `MISSION.md` + `learning-records/` and picks up where
   we left off.
3. Or just say: *"continue my graphic design teaching."*

## What's here
| Folder / file | What it is |
|---|---|
| `MISSION.md` | Why I'm learning this + what success looks like. The compass. |
| `lessons/` | The lessons — open the `.html` files in a browser. Start at `0001`. |
| `reference/glossary.html` | Authoritative glossary of design terms. Print it; keep it handy. |
| `RESOURCES.md` | The trusted external resources behind the lessons. |
| `learning-records/` | What I've actually learned (drives what comes next). |
| `assets/` | Shared styles + the quiz widget the lessons use. |
| `NOTES.md` | Teaching preferences + working notes. |

## Start here
→ Open **`lessons/0001-the-map-of-graphic-design.html`** in your browser.
Then do the Procreate exercise at the end — that's where it sticks.

## Backup
Safe on two fronts: this repo lives on **GitHub** (`amarashi/learning`) and the
folder is **synced to Google Drive**. Sessions get committed & pushed as we go, so
the teaching history is versioned.
